# Project Overview

BoiseCodeCamp.com is the official website for **Boise Code Camp**, a free, community-driven software development technical event in the Northwest. This repository serves as the **production build** and deployment target for the site, likely hosted on GitHub Pages (indicated by the `CNAME` pointing to `boisecodecamp.com`).

The site is built as a single-page application (SPA) using **Vue.js** (v3) and includes integrated services for event management.

## Main Technologies

*   **Framework:** [Vue.js](https://vuejs.org/) (v3) with [Vue Router](https://router.vuejs.org/).
*   **Styling:** [Bootstrap](https://getbootstrap.com/) and custom CSS.
*   **Icons:** [Material Design Icons](https://materialdesignicons.com/).
*   **Content & Scheduling:** Integrates with [Sessionize](https://sessionize.com/) for schedules and speaker management.
*   **Utilities:** Uses `marked.js` for Markdown rendering.

## Directory Structure

*   `/assets`: Minified and hashed production assets (JavaScript, CSS, Images).
*   `/img`: Static image assets, including the Boise Code Camp logo and icons.
*   `/schedule`: Contains a dedicated entry point for the event schedule, embedding a Sessionize grid.
*   `/sponsor-packet`: A sub-section of the site containing sponsorship information and PDF documents for current and past years (2020-2025).
*   `index.html`: The main entry point for the SPA.
*   `manifest.json`: Web app manifest for PWA capabilities.
*   `robots.txt`: Standard crawler instructions.
*   `CNAME`: Domain configuration for `boisecodecamp.com`.

## Building and Running

This repository contains **pre-built production assets**. There is no local build step required for this specific directory.

### Running Locally
To preview the site locally, use any static file server:
```bash
# Example using npx
npx serve .

# Example using Python
python -m http.server 8000
```

### Source Code
The source code for this project (Vue components, source styles, etc.) is likely maintained in a separate repository or branch. The `index.html` file references a build identifier `codecamp_24`, suggesting the project is updated annually.

## Development Conventions

*   **Production Only:** Do not edit files in `/assets` directly as they are generated by a build tool.
*   **Static Content:** Updates to static files like `robots.txt`, `CNAME`, or the PDFs in `sponsor-packet` can be made directly.
*   **Schedule Updates:** The schedule is primarily managed via Sessionize; changes there will reflect on the site through the embedded script in `/schedule/index.html`.
